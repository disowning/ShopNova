# ShopNova 电商系统

ShopNova 是一套基于 React、TypeScript 和 Supabase 构建的电商系统源码，包含用户端商城、商品详情、购物车、结算、模拟支付、订单成功页，以及后台仪表盘、商品管理、分类管理、标签管理、订单管理、支付记录和客户管理等模块。

本项目适合作为电商演示系统、二次开发基础、教学项目或源码交付模板。当前版本以开发测试和功能演示为目标，已接入 Supabase 数据表和模拟支付流程，生产使用前需要替换认证、支付、安全策略和文件存储方案。

## 技术栈

- 前端框架：React 18
- 开发语言：TypeScript
- 构建工具：Vite
- 样式方案：Tailwind CSS
- 图标库：lucide-react
- 数据服务：Supabase
- 本地状态：React Context
- 国际化：项目内置 `src/i18n`，包含中文、英文、日文语言资源

## 功能模块

- 用户端商城：首页、分类、商品列表、商品详情、购物车、结算、订单成功页、登录注册、个人账户
- 商品体系：商品分类、商品多图、SKU、商品参数、商品标签、上下架状态
- 订单体系：收货地址、订单、订单明细、支付记录、SKU 快照
- 后台管理：仪表盘、商品管理、分类管理、标签管理、订单管理、支付记录、客户管理
- 开发测试能力：测试账号、模拟支付、示例商品数据、Supabase 迁移脚本

## 用户端功能

- 首页展示商品分类、精选商品、限时折扣、新品、权益说明、评价区和订阅区。
- 商品列表支持按分类进入，并从 Supabase 商品表读取数据，读取失败时回退到本地 mock 数据。
- 商品详情页支持商品多图、SKU 选择、价格和库存联动、商品参数、亮点信息、相关商品推荐。
- 购物车通过 React Context 在前端内存中维护，支持添加商品、按 SKU 合并、修改数量、移除商品。
- 结算页支持联系人、收货地址、配送方式、支付方式、优惠码、订单备注和订单汇总。
- 登录注册使用自建 `users` 表，登录态保存到 `localStorage`。
- 个人账户页支持查看基础资料和当前用户订单列表、订单详情。

## 后台管理功能

- 仪表盘：订单统计、近 30 天趋势、订单状态分布、订单列表入口。
- 商品管理：商品列表、搜索、分类筛选、状态筛选、新增、编辑、软删除、上下架、图片 URL、SKU、参数、标签和 SEO 字段维护。
- 分类管理：分类列表、搜索、新增、编辑、软删除、启用/停用、父级分类字段。
- 标签管理：标签列表、搜索、新增、编辑、软删除、颜色和状态维护。
- 订单管理：订单列表、搜索、状态筛选、支付状态筛选、日期筛选、分页、订单详情侧栏。
- 订单详情：展示订单、商品、收货地址、支付信息，并支持修改订单状态。
- 支付记录：展示支付流水列表、交易统计、卡号末四位和交易号。
- 客户管理：展示客户列表、搜索、会员等级筛选和客户统计。

## 数据库说明

项目使用 Supabase，数据库迁移位于 `supabase/migrations`。

主要表包括：

- `users`：开发测试用户表，包含邮箱、密码字段、角色、联系方式和客户管理字段。
- `product_categories`：商品分类表。
- `products`：商品主表。
- `product_images`：商品图片表。
- `product_skus`：商品 SKU 表。
- `product_attributes`：商品参数表。
- `product_tags`：商品标签表。
- `product_tag_relations`：商品和标签关联表。
- `shipping_addresses`：收货地址表。
- `orders`：订单主表。
- `order_items`：订单明细表，包含 SKU 快照字段。
- `payments`：模拟支付记录表。
- `risk_orders`：风险订单表，当前属于扩展管理页面使用。

当前迁移中未发现 `carts`、`cart_items`、`coupons` 三张表。购物车由前端内存状态维护，优惠码由前端固定规则计算。

## 模拟支付说明

当前支付流程位于 `src/lib/checkoutService.ts`。

- `simulatePayment` 为开发测试函数。
- 函数会等待短暂延迟后固定返回成功。
- 成功后生成 `SIM-时间戳-随机数` 格式的模拟交易号。
- 订单会写入 `orders` 和 `order_items`。
- 支付记录会写入 `payments`。
- 当前测试模式会保存银行卡号、有效期和 CVV 字段，不能用于生产环境。

## 本地运行步骤

1. 安装依赖：

```bash
npm install
```

2. 创建环境变量文件：

```bash
cp .env.example .env
```

3. 在 `.env` 中填写 Supabase 项目地址和 anon key。

4. 启动开发服务：

```bash
npm run dev
```

5. 浏览器访问 Vite 输出的本地地址。

## 安全说明

当前项目适合开发测试和演示，不建议直接部署为生产系统：

- `users.password_hash` 当前实际保存并比对明文密码，生产环境必须替换为 bcrypt、argon2 或 Supabase Auth。
- 登录态保存在 `localStorage` 的 `shopnova_user` 中，缺少服务端会话校验。
- 多数 Supabase RLS 策略对 `anon` 角色较宽松，方便开发测试，但生产环境必须收紧权限。
- 模拟支付会保存完整卡号、有效期和 CVV 字段，生产环境必须移除敏感字段存储并接入合规支付网关。
- 后台权限主要依赖前端角色判断和自建用户表，生产环境需要服务端权限校验。
- 当前图片以 URL 方式维护，尚未接入安全的文件上传、鉴权和 CDN 存储流程。
