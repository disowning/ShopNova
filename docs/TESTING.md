# ShopNova 手动测试清单

本文档用于开发者或交付验收人员手动验证当前功能。测试前请确认 `.env` 已配置 Supabase，依赖已安装，开发服务已启动。

## 基础准备

- [ ] 执行 `npm install` 安装依赖。
- [ ] 执行 `npm run dev` 启动本地服务。
- [ ] 确认 Supabase 已执行项目迁移。
- [ ] 确认测试账号可用：
  - 普通用户：`customer@test.com / 123456`
  - 管理员：`admin@test.com / 123456`

## 普通用户注册登录

- [ ] 进入前台商城。
- [ ] 打开登录页。
- [ ] 使用 `customer@test.com / 123456` 登录。
- [ ] 预期结果：登录成功，用户菜单或账户入口展示当前用户信息。
- [ ] 退出登录。
- [ ] 进入注册页，使用未注册邮箱填写姓名、手机号和密码注册。
- [ ] 预期结果：注册成功并进入登录态。

## 管理员登录

- [ ] 通过底部模式切换进入管理后台。
- [ ] 使用 `admin@test.com / 123456` 登录。
- [ ] 预期结果：进入后台仪表盘。
- [ ] 使用普通用户账号尝试进入后台。
- [ ] 预期结果：提示没有管理员权限或无法进入后台。

## 后台新增商品

- [ ] 进入后台“商品管理”。
- [ ] 点击添加商品。
- [ ] 在“基本信息”中填写商品名称、分类、简介、描述、状态和展示标记。
- [ ] 在“价格库存”中填写售价、原价、库存、评分、销量。
- [ ] 保存商品。
- [ ] 预期结果：商品保存成功，商品列表出现新商品。

## 添加商品图片

- [ ] 编辑刚创建的商品。
- [ ] 进入“图片管理”标签页。
- [ ] 添加至少两条图片 URL，并填写可选 Alt 文本。
- [ ] 保存商品。
- [ ] 预期结果：商品列表展示第一张图作为主图，前台详情页可看到多图缩略图。

## 添加 SKU

- [ ] 编辑商品并进入“SKU 管理”。
- [ ] 添加至少两个 SKU。
- [ ] 填写 SKU 名称、编码、价格、原价、库存和属性，例如 `颜色: 黑色`、`颜色: 白色`。
- [ ] 保存商品。
- [ ] 预期结果：前台详情页展示 SKU 选项，选择不同 SKU 时价格和库存正确变化。

## 添加商品参数

- [ ] 编辑商品并进入“规格参数”。
- [ ] 添加参数，例如“材质”“尺寸”“保修期”。
- [ ] 保存商品。
- [ ] 预期结果：前台商品详情页参数表展示新增参数。

## 添加商品标签

- [ ] 进入后台“标签管理”。
- [ ] 新增一个标签，填写名称、slug、颜色和描述。
- [ ] 回到“商品管理”，编辑商品。
- [ ] 进入“标签”标签页，勾选刚创建的标签。
- [ ] 保存商品。
- [ ] 预期结果：商品标签关联保存成功。注意：当前前台主要展示 `products.tag` badge，完整多标签筛选仍待完善。

## 前台查看商品详情

- [ ] 切换到用户端商城。
- [ ] 进入商品列表或首页商品卡片。
- [ ] 点击商品进入详情页。
- [ ] 预期结果：展示商品图片、名称、价格、评分、销量、SKU、数量选择、亮点、参数、描述和相关商品。

## 选择 SKU

- [ ] 在商品详情页选择不同 SKU 属性。
- [ ] 预期结果：选项高亮，价格、原价、库存随匹配 SKU 更新。
- [ ] 选择库存为 0 的 SKU。
- [ ] 预期结果：加入购物车按钮不可用或展示缺货状态。

## 加入购物车

- [ ] 选择可售 SKU。
- [ ] 调整数量。
- [ ] 点击加入购物车。
- [ ] 预期结果：购物车抽屉打开，商品按 SKU 展示。
- [ ] 再次加入同商品同 SKU。
- [ ] 预期结果：购物车中同 SKU 数量增加，不新增重复行。
- [ ] 修改数量或删除商品。
- [ ] 预期结果：金额汇总实时更新。

## 结算

- [ ] 保持普通用户登录。
- [ ] 从购物车进入结算页。
- [ ] 填写邮箱、手机号、收货人、国家、省份、城市、邮编、街道地址。
- [ ] 选择配送方式。
- [ ] 可选填写订单备注。
- [ ] 可输入优惠码 `WELCOME15`、`SAVE10` 或 `NOVA20`。
- [ ] 预期结果：订单汇总金额、配送费和优惠金额正确更新。

## 输入任意测试卡号

- [ ] 在支付方式中选择银行卡。
- [ ] 填写持卡人、任意卡号、有效期和 CVV。
- [ ] 预期结果：当前只校验必填字段，不调用真实支付网关。

## 模拟支付成功

- [ ] 点击确认支付。
- [ ] 等待提交完成。
- [ ] 预期结果：模拟支付固定成功，生成 `SIM-...` 交易号，并写入订单和支付记录。

## 查看订单成功页

- [ ] 支付成功后查看订单成功页。
- [ ] 复制订单号。
- [ ] 点击继续购物或返回首页。
- [ ] 预期结果：订单号、支付状态、订单时间线和操作按钮展示正常。

## 后台查看订单详情

- [ ] 切换到后台管理。
- [ ] 进入“订单管理”。
- [ ] 搜索或选择刚创建的订单。
- [ ] 点击查看。
- [ ] 预期结果：右侧订单详情展示订单信息、商品明细、SKU 快照、用户信息、支付信息和收货地址。

## 后台修改订单状态

- [ ] 在订单详情侧栏点击“处理中”。
- [ ] 预期结果：订单状态更新为处理中，列表刷新后状态同步。
- [ ] 继续点击“已发货”“已完成”或“已取消”。
- [ ] 预期结果：订单状态成功更新。

## 支付记录检查

- [ ] 进入后台“支付记录”。
- [ ] 查看刚创建订单的支付流水。
- [ ] 预期结果：展示交易号、订单号、支付方式、卡号末四位、金额、状态和时间。
- [ ] 注意：导出和详情按钮当前未发现完整业务逻辑，不作为通过项。

## 客户管理检查

- [ ] 进入后台“客户管理”。
- [ ] 搜索测试用户邮箱。
- [ ] 切换会员等级筛选。
- [ ] 预期结果：客户列表和统计展示正常。
- [ ] 注意：添加客户和查看按钮当前未发现完整业务逻辑，不作为通过项。

## 构建验证

- [ ] 执行 `npm run build`。
- [ ] 预期结果：构建完成并生成 `dist`。
- [ ] 如提示 `vite` 不存在，先执行 `npm install` 后重新构建。
