# ShopNova 使用说明

本文档给店铺管理员和交付客户使用，说明系统上线后怎么操作。

## 前台商城

- 客户可以浏览首页、分类页、商品列表和商品详情。
- 商品详情支持多图、SKU、库存联动、数量选择、加入购物车、评论和相关商品。
- 客户进入结算前需要登录。
- 结算页支持联系人、收货地址、配送方式、优惠码、支付方式、订单备注和订单汇总。
- 开发测试支付只用于测试流程，不会真实扣款。测试时不要填写真实银行卡信息。
- Stripe / PayPal 开启后，会通过 Supabase Edge Functions 创建真实支付会话并跳转支付。

## 后台管理

管理员登录后可以使用这些模块：

- 仪表盘：查看订单、收入、支付状态、订单趋势等统计。
- 订单管理：搜索、筛选、查看详情、修改订单状态。
- 商品管理：新增、编辑、上下架、软删除商品，维护图片、价格、库存、SKU、参数、标签、SEO 和商品评论。
- 分类管理：维护商品分类、父级分类、排序和状态。
- 标签管理：维护商品标签、颜色和状态。
- 图片管理：维护后台可用媒体资源。
- 内容管理：维护前台可编辑文案和页面内容。
- 翻译管理：维护多语言文本、JSON 导入导出和 AI 翻译。
- 站点配置：维护站点名称、导航、页脚、首页模块、信任背书等。
- 支付管理：管理开发测试支付、Stripe、PayPal、支付密钥和支付流水。
- 客户管理：新增客户、查看客户订单、编辑资料、重置密码、封禁/恢复、删除无订单客户、配置会员规则。
- 风险订单：查看和审核高风险订单。
- 数据分析：按已支付有效订单查看营收、订单趋势和产品排行。
- 系统设置：维护管理员资料、通知偏好、密码和 Google 登录配置。

## 支付管理

开发测试支付不要删除，它用于测试下单流程。客户不需要时可以在后台关闭。

真实支付上线前需要：

1. 完成 Supabase 数据库迁移。
2. 部署支付相关 Edge Functions。
3. 设置 Edge Function secrets。
4. 在后台支付管理中启用 Stripe 或 PayPal。
5. 保存公开配置和真实密钥。
6. 使用支付平台 test mode 完整测试一次下单和回调。

支付密钥不要写到前端源码。

## Google 登录

后台操作：

1. 打开 `系统设置`。
2. 进入 `第三方登录`。
3. 开启 Google 登录。
4. 填写 Google Web Client ID，例如 `xxxx.apps.googleusercontent.com`。
5. 可选填写 Workspace 域名限制，例如 `example.com`。
6. 建议保持“自动创建客户账号”开启。
7. 保存设置。

Google Cloud Console 也要把商城域名加入 Authorized JavaScript origins。

## 日常运营

- 每天先看仪表盘、订单管理和风险订单。
- 修改商品后，到前台确认价格、库存、SKU 和图片是否正确。
- 修改内容或站点配置后，到前台确认展示是否生效。
- 改支付配置后，用开发测试支付或支付平台 test mode 完整下单一次。
- 如果历史客户统计不准，到客户管理中点击同步历史统计。
- 交付客户前，修改默认管理员和测试客户密码。
